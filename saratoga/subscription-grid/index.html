subscription grid

The subscription pages have a custom grid, and an accompanying column concept, to allow for maximum control over the order of inputs at any screen width. These rules are based on the full media query being relevant. In the case that this content is going in a modal appended to the DOM of a larger page, we will need to adjust.

The .grid class hard-codes a one-column or two-column layout depending on the screen width. The .column class can be used as one of the grid children to force smaller inputs.

EXAMPLE

Use your email address or a social account

Error text replaces help.

Must be at least 7 characters long

HTML

<div class="package">
  <h4 class="sans bold">Use your email address or a social account</h4>
  <div class="grid">
    <div class="two columns">
      <div class="input">
        <label for="firstname">First name</label>
        <input name="firstname" type="text">
        <p class="small"></p>
      </div>
      <div class="input">
        <label for="lastname">Last name</label>
        <input name="lastname" type="text">
        <p class="small error show">Error text replaces help.</p>
      </div>
    </div>
    <div class="input">
      <label for="email">Email address</label>
      <input name="email" type="text">
      <p class="small"></p>
    </div>
    <div class="input">
      <label for="password">Password</label>
      <input name="password" type="password">
      <p class="small show">Must be at least 7 characters long</p>
    </div>
  </div>
</div>

CSS

.grid {
  grid-template-columns: 1fr;
}

.package .grid {
  grid-column-gap: 15px;
  grid-row-gap: 10px;
}

.columns {
  display: grid;
  grid-gap: 15px;
}

.columns.two {
  grid-template-columns: repeat(2, 1fr);
}

.columns.four {
  grid-template-columns: repeat(4, 1fr);
}

.span-2 {
  grid-column: span 2;
}

@media (min-width: 545px) {
  .grid {
    grid-template-columns: 1fr 1fr;
  }

  .columns {
    grid-column: span 2;
  }
}
